# Документація до проєкту Cinema API

Даний проєкт є серверною частиною (REST API) системи бронювання квитків у кінотеатр. Розробка виконана в межах Лабораторної роботи №1 з дисципліни «Архітектура програмного забезпечення».

## Технологічний стек
* **Node.js, Express.js** — основна платформа та фреймворк сервера.
* **PostgreSQL (pg)** — система управління базами даних.
* **JWT (jsonwebtoken)** — механізм авторизації та розмежування прав доступу.
* **Jest, Supertest** — середовище для автоматизованого тестування компонентів.

## Інструкція з розгортання проєкту

### 1. Системні вимоги
* Середовище виконання Node.js.
* Встановлений сервер бази даних PostgreSQL.

### 2. Підготовка бази даних
1. Створіть нову базу даних у PostgreSQL.
2. Для створення структури таблиць (користувачі, фільми, сеанси, бронювання) та встановлення зв'язків між ними виконайте SQL-запити з файлу `database/init.sql`.

### 3. Конфігурація середовища
Для коректної роботи додатка необхідно створити файл `.env` у кореневій директорії проєкту та вказати параметри підключення. Приклад вмісту файлу:

```env
PORT=3000
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=1365244
DB_NAME=cinema
```

### 4. Встановлення та запуск
Виконайте наступні команди в терміналі:

```
# Інсталяція необхідних модулів
npm install
# Запуск сервера
npm start
```
Після запуску API буде доступне за адресою: http://localhost:3000.

## Тестування системи
У проєкті реалізовано модульне (Unit) та інтеграційне тестування для перевірки логіки валідації даних та роботи механізмів захисту маршрутів (Middlewares).

Запуск усіх тестів виконується командою:
```
npm test
```
